기획서: 라틴댄스(바차타/살사) 자동 카운트 & 안무 큐 앱
1) 한 줄 요약

바차타·살사 음악(오디오/영상)을 분석해 자동 카운트(1–4 / 1–8)를 안정적으로 제공하고, 시각화 + 클릭/삐/음성 큐로 안무 타이밍을 잡아주는 연습용 플레이어.
유튜브는 “많이 쓰는 재생원”이라는 현실을 반영해 오버레이(참조) 모드 + 정식 구매 유도를 함께 제공.

2) 문제 정의

취미 댄서들은 유튜브로 음악을 틀어놓고 연습하지만,

카운트를 정확히 잡기 어렵고

안무 포인트(턴/브레이크/체인지) 타이밍을 공유/반복하기가 불편함.

자동 카운트가 핵심인데, 유튜브 스트리밍은 기술/정책 제약으로 “원시 오디오 분석”이 막힐 가능성이 높음 → 제품 설계를 “유튜브만”에 걸면 치명적.

3) 타깃 사용자

바차타/살사 취미 댄서(개인 연습)

강사/팀(안무 제작·리허설)

학생(강사 영상으로 복습, 촬영한 연습 영상으로 교정)

4) 핵심 가치 제안(Value Proposition)

자동 카운트: 음악만 있으면 즉시 1–4/1–8 카운트가 따라감(특히 파일/영상에서 고정밀).

안무 큐: “5에서 턴”, “1에서 브레이크” 같은 타이밍 힌트를 클릭/삐/음성으로 자동 안내.

학습 보조: 악기 강조(타악/베이스/고역)로 리듬을 더 잘 들리게.

현실 대응: 유튜브 기반 연습 문화는 오버레이 모드로 최대한 지원하고, 필요하면 정식 구매(다운로드)로 풀 기능 유도.

5) 제품 모드(핵심 설계)
A. 로컬 파일 모드(핵심/정답)

입력: mp3/wav/flac 등 오디오 파일, 또는 mp4/mov 등 영상 파일(오디오 트랙 포함)

제공:

자동 BPM/비트/다운비트 추정 → 완전 자동 카운트

큐(삐/클릭/음성/TTS) + 시각화 + 루프 연습

악기 강조(기본 EQ)

포지션: “정확도/기능/저작권 리스크” 모두에서 최우선 모드

B. 로컬 연습 영상 모드(학생 촬영/강사 파일) — 강력한 차별화

입력: 학생이 촬영한 연습 영상, 강사가 준 영상 파일

제공:

영상 재생 위에 카운트 오버레이

음악이 녹음되어 있으면 자동 카운트 가능(음질 조건 충족 시)

강사 멘트 구간 등으로 흔들리면 “원탭 재동기화(지금이 1)” 제공

포지션: 현실 사용성 최고(실제 학원/수업 환경에 맞음)

C. 유튜브 오버레이 모드(참조/보조)

입력: 유튜브 링크(또는 내장 플레이어)

제공(현실적으로 가능한 범위):

카운트 표시 + 큐 안내(재생 시간 기준)

자동은 제한적이므로 **탭 템포 + ‘지금이 1’**로 빠르게 맞추는 반자동이 기본

(옵션) 마이크 기반 자동 시도(환경에 따라 정확도 편차 큼)

포지션: “유튜브로 연습하는 사람을 못 버린다”를 해결하는 최소한의 해법

6) 카운트 규칙(댄스 UX 기준)

바차타: 8카운트 기반. step(1)–step(2)–step(3)–TAP(4)–step(5)–step(6)–step(7)–TAP(8)
  - 4/4 박자 × 2마디 = 8카운트 1사이클
  - 1,2,3 = 사이드 스텝, 4 = 탭 + 힙 액센트
  - 5,6,7 = 반대 방향 스텝, 8 = 탭 + 힙 액센트
  - 음악 구조: Derecho(절) / Majao(후렴) / Mambo(브릿지)

살사: 8카운트 기반. 1–2–3–(4 쉼) 5–6–7–(8 쉼)
  - 핵심 차이: 바차타는 4,8에 탭(움직임 있음), 살사는 4,8이 쉼(pause)

on1/on2 선택(표기/큐 템플릿만이라도 제공)

7) 핵심 기능 정의
7.1 자동 분석(파일/영상에서 “풀 기능”)

BPM/비트 위치(그리드)

다운비트(1) 추정

오프셋/드리프트 보정

실패 대비 UX: “지금이 1” 원탭 보정 + 미세 이동

7.2 시각화 UI

큰 카운터(현재 1~4/1~8)

비트 펄스(1 강/나머지 약)

타임라인 비트 마커 + 8카운트/마디 경계 표시

연습 루프(구간 반복)

7.3 큐 시스템(안무 지원)

큐 타입: 클릭/삐 / 음성 샘플 / TTS(“원-투-쓰리…”, “5에 턴”)

큐 배치:

수동 배치(비트/카운트 기준)

템플릿 배치(예: 8카운트마다 안내, 16카운트마다 변형)

프리셋 저장/공유(곡별/안무별)

7.4 악기 강조(학습 보조)

1차: EQ 기반(베이스/타악/고역) + 트랜지언트 강조

2차: 고성능 환경에서 소스 분리(스템) 기반 강조(옵션)

8) 유튜브 대응 정책(제품 정책으로 명확히)

유튜브 링크는 참조(재생/오버레이)용으로 지원

고정밀 자동 카운트·악기 강조·내보내기 등 핵심 기능은 로컬 파일에서 제공

사용자가 “원곡을 구매해서 쓰고 싶다”면

다운로드 판매처(정식 구매 링크)로 유도

구매 후 파일을 가져오면 풀 기능 사용 가능

앱은 유튜브 콘텐츠를 추출/다운로드/변환하는 기능을 제공하지 않음(리스크 차단)

9) AI 생성 트랙(아이디어 확장, 단 “복제” 금지)
목적

구매본/파일이 없을 때도 연습 가능하도록 오리지널 연습 트랙 제공

제공 형태

사용자가 원하는 조건(BPM, 에너지, 악기 구성, 8카운트 구조)로 완전히 새 곡 생성

카운트/프레이즈/브레이크 포인트 메타데이터를 같이 제공(큐 자동화와 궁합)

안전 원칙(필수)

특정 곡/아티스트를 “아주 유사하게” 만드는 기능은 제공하지 않음

생성 프롬프트는 곡명/아티스트 기반이 아니라 댄스 조건 기반으로만 구성

10) 콘텐츠/세션 관리

라이브러리: 트랙(파일/영상/유튜브 링크), 안무 프리셋, 큐 프리셋

저장 항목:

BPM, 오프셋, 다운비트 보정값

큐 타임라인

악기 강조 설정

내보내기(선택):

큐 리스트(텍스트/이미지)

연습 스크립트(카운트 기반 안내문)

11) 플랫폼 범위

모바일: iOS/Android (연습/촬영 영상/현장 사용 중심)

데스크톱: macOS/Windows (편집/안무 설계/수업 준비 중심)

12) 수익 모델(현실적인 구성)

Freemium:

무료: 기본 카운트/기본 큐, 제한된 저장 슬롯

유료 구독: 무제한 저장, 고급 큐(템플릿/공유), 악기 강조 고급, 생성 트랙 크레딧(옵션)

팀/강사용 플랜:

프리셋 공유, 클래스별 세트리스트, 학생 배포용 링크/파일 패키지

13) 성공 지표(KPI)

파일/영상 모드에서:

자동 카운트 정확도(사용자 수정 없이 연습 가능한 비율)

“지금이 1” 보정 후 안정화율

유튜브 모드에서:

탭 템포/원탭 싱크 완료율(초기 세팅 이탈률 감소)

리텐션:

저장 프리셋 재사용률

루프 연습 사용 시간

전환:

구매 유도 클릭률(정식 판매처 이동)

유료 구독 전환율

14) 리스크 및 대응(기획 단계에서 박아둘 것)

다운비트(1) 자동 인식은 100% 불가능 → 보정 UX가 제품의 일부여야 함

유튜브는 자동 분석이 불안정 → 오버레이/반자동 중심으로 기대치를 통제

연습 영상은 잡음/멘트로 흔들림 → 재동기화 버튼/구간별 신뢰도 표시 같은 장치 필요

생성 트랙은 “유사 복제” 요구가 들어옴 → 제품 정책/가드레일을 UI/문구/기능 제한으로 강하게 적용

15) MVP 범위(기획 확정안)

**MVP는 “로컬 파일/영상에서 완전 자동 카운트 + 큐 + 저장”**으로 성공시키고,
유튜브는 오버레이(탭 템포 + 원탭 싱크)까지만 넣는다.
AI 생성 트랙은 MVP 이후(또는 베타)로 두되, 오리지널 연습 트랙으로만 제공.
