## 🚀 출시 전 체크리스트

### 1. 앱 품질 점검
- [ ] 전체 코드베이스 하드코딩 문자열 최종 스캔 (i18n 누락)
- [ ] 크래시 없이 전체 플로우 테스트 (로그인 → 라이브러리 → 플레이어 → 커뮤니티)
- [ ] 오프라인 상태에서 기본 기능 작동 확인
- [ ] 메모리 누수 확인 (오디오/비디오 재생 반복)
- [ ] 다양한 화면 크기 테스트 (소형 폰 ~ 태블릿)
- [ ] 다크모드/라이트모드 확인 (현재 다크 고정이면 OK)
- [ ] 10개 언어 각각 UI 깨짐 없는지 확인 (긴 텍스트 잘림 등)

### 2. app.json / app.config 설정
- [ ] `name`: 스토어 표시 이름 확정 ("Musicality")
- [ ] `slug`: URL-safe 이름 ("musicality")
- [ ] `version`: "1.0.0" 설정
- [ ] `icon`: 1024×1024 앱 아이콘 (PNG, 투명 배경 없이)
- [ ] `splash`: 스플래시 이미지 설정 (1284×2778 권장)
- [ ] `adaptiveIcon`: Android용 아이콘 (foreground + background)
- [ ] `bundleIdentifier` (iOS): 예) "com.ongo.musicality"
- [ ] `package` (Android): 예) "com.ongo.musicality"
- [ ] `permissions`: 필요한 권한만 명시 (AUDIO, READ_EXTERNAL_STORAGE 등)
- [ ] `privacy`: 개인정보 처리 URL

### 3. 에셋 준비
- [ ] 앱 아이콘 (1024×1024) — 둥근 모서리 자동 적용됨
- [ ] 스플래시 스크린 이미지
- [ ] 스토어 스크린샷 (최소 2장, 권장 5~8장)
  - Android: 폰 (1080×1920 이상), 태블릿 (선택)
  - iOS: 6.7" (1290×2796), 6.5" (1284×2778), 5.5" (1242×2208)
- [ ] 피처 그래픽 (Android, 1024×500)
- [ ] 프로모션 텍스트 / 짧은 설명 (80자)
- [ ] 상세 설명 (4000자 이내, 다국어)
