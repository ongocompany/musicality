# 문제 분석 & 개선 로드맵

## "1을 찾는" 문제 — 근본 원인

앱의 핵심 기능은 "1박 자동 탐지". 현재 문제:

1. **Madmom 다운비트가 바차타/살사에 부정확** — 서양 팝/록 학습 데이터 편향
2. **Librosa 구조 분석이 구식** — derecho 감지 recall 16%, mambo/majao 혼동
3. **에너지 분기점 → 춤 변곡점 연결 로직 부재** — 퍼커션 패턴 변화를 섹션으로 매핑 못함

## 개선 로드맵

### 단기 (현재 인프라에서)
- [ ] **Demucs 퍼커션 분리** → 현재 엔진에 적용
  - 퍼커션 트랙만 분리 → Madmom에 입력 → 다운비트 정확도 개선 기대
- [ ] **구조 분석 v3**: 퍼커션 에너지 프로파일 기반 섹션 분류
- [ ] **배치 파이프라인 완료**: 500곡+ 고신뢰도 데이터셋 확보

### 중기 (ML 파인튜닝)
- [ ] **Beat This! 파인튜닝 v2**: 500곡+ 데이터, augmentation, LR 0.00005
- [ ] **인간 라벨 수집**: pnote 포맷으로 수동 비트 라벨링
- [ ] **라틴 룰 엔진**: 클라베 패턴, 콩가 패턴 등 장르 특화 규칙

### 장기 (통합)
- [ ] **퍼커션 분리 + 파인튜닝 모델 + 라틴 룰 엔진** 결합
- [ ] Madmom 대비 정확도 동등 + 10배 속도 목표
